# بازی نبرد ناوها

یک پروژه ساده و کامل از بازی دو نفره `Battleship` با پایتون، `WebSocket` و رابط کاربری تحت وب. این پروژه هم سرور بازی را اجرا می‌کند، هم فایل‌های فرانت‌اند را به صورت محلی در اختیار مرورگر قرار می‌دهد. دو بازیکن به سرور وصل می‌شوند، کشتی‌های خود را روی صفحه `10x10` می‌چینند و سپس به صورت نوبتی به خانه‌های حریف شلیک می‌کنند.

رابط کاربری و پیام‌های بازی فارسی هستند و منطق بازی روی سرور مدیریت می‌شود.

## ویژگی‌ها

- بازی دو نفره هم‌زمان با `WebSocket`
- صفحه بازی تحت وب بدون نیاز به فریم‌ورک فرانت‌اند
- چیدمان مرحله‌ای کشتی‌ها با اعتبارسنجی کامل
- مدیریت نوبت بازیکنان روی سرور
- ثبت امتیاز و تشخیص برنده
- ثبت لاگ رویدادهای بازی در پوشه `logs/`
- راه‌اندازی سریع با اسکریپت `run.sh`

## تکنولوژی‌ها

- `Python 3`
- کتابخانه [`websockets`](https://pypi.org/project/websockets/)
- `HTML`, `CSS`, `JavaScript`
- `asyncio` برای مدیریت هم‌زمان اتصال‌ها

## ساختار پروژه

```text
battleship/
├── game_engine.py      # منطق اصلی بازی، صفحه‌ها، کشتی‌ها و امتیاز
├── server.py           # سرور HTTP و WebSocket
├── requirements.txt    # وابستگی‌های پایتون
├── run.sh              # اسکریپت اجرای سریع پروژه
├── static/
│   ├── index.html      # رابط کاربری بازی
│   ├── app.js          # منطق سمت کاربر و ارتباط با سرور
│   └── style.css       # ظاهر رابط کاربری
└── logs/               # لاگ‌های اجرا
```

## قوانین بازی

- صفحه بازی `10x10` است.
- سطرها با `A` تا `J` و ستون‌ها با `1` تا `10` مشخص می‌شوند.
- هر بازیکن باید 5 کشتی زیر را بچیند:
  - `Carrier` با طول 5
  - `Battleship` با طول 4
  - `Cruiser` با طول 3
  - `Submarine` با طول 3
  - `Destroyer` با طول 2
- کشتی‌ها فقط می‌توانند افقی یا عمودی باشند.
- قرار دادن کشتی به صورت مورب مجاز نیست.
- کشتی‌ها نباید روی هم بیفتند.
- هر اصابت `1` امتیاز دارد.
- مجموع خانه‌های تمام کشتی‌ها `17` است، بنابراین بازیکنی که به امتیاز `17` برسد برنده بازی می‌شود.
- اگر شلیک بازیکن به هدف بخورد، همان بازیکن دوباره نوبت دارد.
- اگر شلیک خطا باشد، نوبت به حریف منتقل می‌شود.

## نحوه اجرا

### پیش‌نیاز

- `Python 3.10+` پیشنهاد می‌شود
- `python3 -m venv` باید در سیستم در دسترس باشد

### اجرای سریع

```bash
chmod +x run.sh
./run.sh
```

این اسکریپت:

1. در صورت نبودن `venv` آن را می‌سازد.
2. وابستگی‌ها را از `requirements.txt` نصب می‌کند.
3. سرور را اجرا می‌کند.

### اجرای دستی

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python server.py
```

## آدرس‌های پروژه

بعد از اجرا، دو سرویس بالا می‌آیند:

- رابط وب: `http://localhost:8080`
- وب‌سوکت بازی: `ws://localhost:8765`

برای شروع:

1. مرورگر را باز کنید و به `http://localhost:8080` بروید.
2. در دو تب یا دو مرورگر جداگانه همان صفحه را باز کنید.
3. هر دو بازیکن به آدرس وب‌سوکت پیش‌فرض `ws://localhost:8765` وصل شوند.
4. پس از ورود بازیکن دوم، مرحله چیدمان کشتی‌ها آغاز می‌شود.

## روند بازی

### 1. لابی

- بازیکن اول وارد حالت انتظار می‌شود.
- با ورود بازیکن دوم، بازی از حالت انتظار خارج می‌شود.
- اگر بازیکن سومی وصل شود، سرور اتصال او را رد می‌کند.

### 2. چیدمان کشتی‌ها

- سرور به ترتیب از هر بازیکن می‌خواهد کشتی‌ها را قرار دهد.
- ترتیب کشتی‌ها ثابت است:
  - `Carrier`
  - `Battleship`
  - `Cruiser`
  - `Submarine`
  - `Destroyer`
- کاربر با انتخاب خانه شروع و خانه پایان، موقعیت کشتی را مشخص می‌کند.
- دکمه `چرخش` برای انتخاب جهت افقی/عمودی در رابط وجود دارد.

### 3. حمله

- بعد از کامل شدن چیدمان هر دو بازیکن، بازی شروع می‌شود.
- بازیکن 1 شروع‌کننده است.
- بازیکن فقط در نوبت خود می‌تواند روی صفحه حریف شلیک کند.
- شلیک تکراری به یک خانه نامعتبر است و با پیام خطا رد می‌شود.

### 4. پایان بازی

- وقتی امتیاز یک بازیکن به `17` برسد، بازی تمام می‌شود.
- برنده به هر دو بازیکن اعلام می‌شود.
- اگر یکی از بازیکنان قطع شود، بازی پایان می‌یابد و وضعیت بازی ریست می‌شود.

## معنی نمادها در صفحه

### نقشه خودی

- `&` خانه‌ای که کشتی شما در آن قرار دارد
- `*` خانه‌ای از شما که حریف به آن شلیک کرده است
- `.` خانه خالی

### نقشه حمله

- `#` خانه‌ای که شما قبلاً به آن شلیک کرده‌اید
- `.` خانه‌ای که هنوز هدف قرار نداده‌اید

## معماری کلی

### `game_engine.py`

هسته منطق بازی در این فایل قرار دارد:

- تعریف اندازه صفحه و لیست کشتی‌ها
- تبدیل مختصات متنی مثل `A1` به اندیس سطر و ستون
- اعتبارسنجی موقعیت کشتی‌ها
- ثبت شلیک‌ها و اصابت‌ها
- محاسبه امتیاز
- ساخت نمای مناسب برای هر بازیکن

کلاس‌های اصلی:

- `Board`
  - اطلاعات کشتی‌های بازیکن
  - خانه‌های اشغال‌شده
  - شلیک‌های دریافت‌شده از حریف
  - شلیک‌های انجام‌شده توسط خود بازیکن
  - امتیاز بازیکن
- `Game`
  - وضعیت کلی بازی
  - نوبت فعلی
  - برنده
  - صفحه هر دو بازیکن
  - ترتیب چیدمان کشتی‌ها

### `server.py`

این فایل دو سرویس را هم‌زمان اجرا می‌کند:

- یک سرور HTTP ساده روی پورت `8080` برای ارسال فایل‌های پوشه `static`
- یک سرور `WebSocket` روی پورت `8765` برای مدیریت منطق بازی

وظایف اصلی سرور:

- پذیرش حداکثر دو بازیکن
- تخصیص شماره بازیکن `1` یا `2`
- مدیریت مراحل `waiting`, `placement`, `playing`, `finished`
- بررسی صحت پیام‌های دریافتی
- ارسال به‌روزرسانی وضعیت به بازیکنان
- ثبت رخدادها در فایل لاگ

### `static/app.js`

منطق سمت کاربر در این فایل قرار دارد:

- اتصال به `WebSocket`
- دریافت وضعیت‌ها و پیام‌های سرور
- رسم دو صفحه بازی
- مدیریت کلیک برای چیدمان کشتی‌ها
- ارسال شلیک و درخواست‌های چیدمان
- نمایش برد یا باخت

### `static/index.html`

شامل:

- صفحه لابی
- نمایش وضعیت اتصال
- نمایش امتیازها
- دو صفحه بازی
- پنل چیدمان کشتی
- پنجره پایان بازی

### `static/style.css`

ظاهر رابط کاربری، رنگ‌بندی خانه‌ها، وضعیت اتصال، صفحه پایان و چیدمان کلی صفحه را کنترل می‌کند.

## پروتکل پیام‌ها

سرور و کلاینت از پیام‌های JSON استفاده می‌کنند.

### پیام‌های ارسالی از کلاینت

قرار دادن کشتی:

```json
{
  "type": "placement",
  "ship": "Carrier",
  "head": "A1",
  "tail": "A5"
}
```

شلیک:

```json
{
  "type": "fire",
  "target": "B7"
}
```

### پیام‌های مهم ارسالی از سرور

- `status`
- `state`
- `placement_request`
- `game_start`
- `fire_result`
- `turn`
- `game_over`
- `error`

نمونه وضعیت:

```json
{
  "type": "state",
  "player": 1,
  "phase": "playing",
  "yourScore": 3,
  "opponentScore": 1,
  "currentTurn": 1,
  "isYourTurn": true,
  "ownGrid": [],
  "attackGrid": [],
  "nextShip": null,
  "shipsPlaced": ["Carrier", "Battleship"],
  "winner": null
}
```

## لاگ‌ها

با هر اجرا، یک فایل لاگ جدید در پوشه `logs/` ساخته می‌شود:

```text
logs/game_YYYYMMDD_HHMMSS.log
```

در این لاگ‌ها موارد زیر ثبت می‌شود:

- اتصال و قطع اتصال بازیکنان
- چیدمان کشتی‌ها
- پیام‌های دریافتی
- نتیجه شلیک‌ها
- شروع و پایان بازی

## نکات توسعه

اگر بخواهید پروژه را توسعه دهید، چند نقطه مناسب برای شروع این‌ها هستند:

- افزودن نام بازیکن‌ها به جای شماره `1` و `2`
- نمایش دقیق‌تر اصابت و خطا روی صفحه حمله
- جلوگیری از نمایش پیش‌نمایش نامعتبر خارج از مرز صفحه به شکل بهتر
- افزودن دکمه شروع مجدد بدون قطع اتصال
- افزودن تست برای منطق `game_engine.py`
- افزودن پشتیبانی از استقرار روی شبکه محلی با آدرس سرور قابل تنظیم
- بهبود سرور HTTP با `Content-Type` کامل‌تر برای فایل‌های استاتیک

## محدودیت‌های فعلی

- بازی فقط از دو بازیکن هم‌زمان پشتیبانی می‌کند.
- داده‌ها در حافظه نگه‌داری می‌شوند و با خاموش شدن سرور از بین می‌روند.
- احراز هویت یا سیستم کاربری وجود ندارد.
- سرور HTTP بسیار ساده است و فقط برای فایل‌های همین پروژه مناسب است.
- تست خودکار برای پروژه تعریف نشده است.

## رفع اشکال

اگر صفحه باز نمی‌شود:

- بررسی کنید `python server.py` یا `./run.sh` هنوز در حال اجرا باشد.
- مطمئن شوید پورت `8080` توسط برنامه دیگری اشغال نشده باشد.

اگر اتصال بازی برقرار نمی‌شود:

- بررسی کنید آدرس `ws://localhost:8765` درست باشد.
- مطمئن شوید پورت `8765` باز و در دسترس باشد.

اگر بازیکن دوم وارد نمی‌شود:

- سرور فقط دو اتصال هم‌زمان را می‌پذیرد.
- اگر اتصال قبلی درست بسته نشده، یک‌بار سرور را ری‌استارت کنید.

## اجرای پیشنهادی برای ارائه

برای نمایش پروژه در کلاس یا دمو:

1. سرور را با `./run.sh` اجرا کنید.
2. دو تب مرورگر باز کنید.
3. در هر دو تب به سرور متصل شوید.
4. چیدمان کشتی‌ها را انجام دهید.
5. چند شلیک موفق و ناموفق انجام دهید تا نوبت، امتیاز و پایان بازی دیده شود.
6. فایل‌های لاگ تولیدشده در پوشه `logs/` را هم برای نشان دادن ثبت رویدادها نمایش دهید.
